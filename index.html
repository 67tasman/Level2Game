Fun Game: Escape the Dungeon

by: Arielle Sanford

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I created this game because I wanted to make something with nothing limiting creativity. I wanted visuals, music, and a plot, so I could attempt to make the game sort of aesthetic. It's harder to be creative with a game like pong.

2. What I Learned

I added a instruction screen because for those who never play computer games, using arrow keys or WASD may not be obvious. I added sound effects as well, because it adds another component to the game which makes it more interesting.

3. Architecture of My Game

I created classes which implemented the 4 screens: the start screen, player select screen, and 2 game screens. The basic setup of a zone(created in the class GameScreen) is a 10 by 10 array of labels overlaid on a 10 by 10 array of panels. In the class Space, I have a constructor that allows me to add several attributes to each panel in the grid(ex. image, color, passibility, state). In the class Instruction, theres a constructor that determines what happens when the player interacts with the panel.(ex. state(trigger), panel affected, new picture, new color, new passibility, new state, warp?, newY, newX, sound effect). In the Zone class I uploaded several image and sound files that are implemented in the game in other classes.

4. Technical Implementation

First I created three screens: a start screen, a player select screen, and the game screen(there are two but they extend from the same class). The game screens are 10 by 10 arrays of labels overlaid on a 10 by 10 array of panels. I am most proud of my space and instruction classes, which assign several attributes to each panel; They are what make up my game, implementing all the visuals, sound effects, and allowing the panels to have different reactions when the player interacts with them. I drew out on paper how I wanted each game screen to look. Then I went through the fun process of finding images for my game. Lastly I added music and sound effects.

5. Most Challenging Part

The most evident challenge was not being able to make the label fill the entire panel, despite hours of efforts to prevent this. As A result each panel has a small gap and the picture is cut off. This is most noticeable on the fountain in the second game screen. Additionally, I was not able to fully figure out the border layout so I have a very simple start screen. Lastly, GitHub repeatedly deleted recent work. I "overcame" this by choosing not having an end screen due to frustrations.

6. Future Fixes and Updates

In the future, I may add more levels, as I originally created the code so it would by easy to implement several levels. Another idea I had that never got carried out was having an AI security guard that chases the player around the palace room. Lastly, I could make the game look more professional by having a more consistent graphics theme.
